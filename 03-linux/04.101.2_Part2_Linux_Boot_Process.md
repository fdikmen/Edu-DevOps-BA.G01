## Boot Logları

### Kernel Ring Buffer ve dmesg
 - Ring buffer içeriğini görüntülemek için `dmesg` komutu kullanılır
 - Yönetici izni gerekebilir, ayrica sayfa sayfa görüntülemek icin -less kullanilabilir
 ```code
 sudo dmesg | less
 ```
 less=> q cikarsiniz, sola-saga okla sayfalar arasinda gezerisniz

 - dmesg filtreleme yapamk icin
 ```code
 sudo dmesg --level=err,warn
 ```
  ```code
 sudo dmesg -T
 sudo dmesg -w
 ```

### Log Dosyaları: /var/log
syslog daemon => /var/log

  | Dosya / Konum        | Açıklama |
  |----------------------|----------|
  | `/var/log/dmesg`     | Kernel ring buffer'ın boot sonrası kaydedilmiş hali (bazı sistemlerde) |
  | `/var/log/boot.log`  | Red Hat tabanlı sistemlerde (Fedora, RHEL, CentOS) boot logu |
  | `/var/log/boot`      | Debian/Ubuntu tabanlı sistemlerde boot logu |
  | `/var/log/messages`  | Red Hat tabanlı sistemlerde genel sistem mesajları |
  | `/var/log/syslog`    | Debian/Ubuntu tabanlı sistemlerde genel sistem mesajları |

  `ls -ltr /var/log`

### journalctl — Systemd Journal
| Araç / Dosya              | Ne Zaman Kullanılır? |
|---------------------------|----------------------|
| dmesg                     | Kernel boot mesajlarını ve donanım tanıma loglarını görmek için |
| journalctl -k            | Kernel loglarını systemd journal üzerinden görüntülemek için |
| journalctl -b            | Mevcut boot'a ait tüm logları görmek için |
| journalctl -xe           | Bir hatadan hemen sonra hızlı tanılama yapmak için |
| journalctl -u <servis>   | Belirli bir servise ait logları filtrelemek için |
| /var/log/boot.log        | Boot sürecinin kalıcı kaydını okumak için (Red Hat tabanlı) |
| /var/log/messages        | Genel sistem mesajlarını incelemek için (Red Hat tabanlı) |

## Init Sistemleri => PID1
init process'in sorumlulukları:
  -	Web sunucusu, SSH, NTP gibi sistem servislerini başlatmak
  -	Servislerin başlatılma sırasını ve bağımlılıklarını yönetmek
  -	Servis çökmesi durumunda yeniden başlatma işlemini gerçekleştirmek
  -	Sistem kapatılırken tüm servisleri düzenli şekilde durdurmak

| Init Sistemi        | Açıklama |
|--------------------|----------|
| SysVinit (SysV)    | Unix System V'den gelen geleneksel init. Script tabanlı, sıralı başlatma. Artık büyük dağıtımlarda kullanılmıyor. |
| Upstart            | Canonical (Ubuntu) tarafından 2007'de geliştirilen event-driven init. Systemd'nin yaygınlaşmasıyla 2015'te terk edildi. |
| systemd            | Günümüzde tüm büyük dağıtımlarda kullanılan modern init sistemi. Paralel başlatma, entegre log, kapsamlı servis yönetimi sunar. |

## systemd
  -	Servisleri **paralel** olarak başlatır → daha hızlı boot
  -	Servisler arası **bağımlılıkları** otomatik yönetir
  -	Kapsamlı ve yapılandırılmış **log sistemi** (journald) sunar
  -	Donanım bağlantısına, zamanlayıcıya veya başka bir servisin durumuna göre servis tetikleyebilir

## systemd Üniteleri (Units)

Birim dosyaları şu dizinlerde aranır (öncelik sırasına göre):
| Dizin                      | Açıklama |
|---------------------------|----------|
| /etc/systemd/system/      | Sistem yöneticisi tarafından oluşturulan veya değiştirilen UNIT dosyaları. En yüksek önceliğe sahiptir; burada yapılan değişiklikler diğerlerini ezer. |
| /run/systemd/system/      | Çalışma zamanı (runtime) sırasında oluşturulan geçici UNIT dosyaları. Sistem yeniden başlatıldığında silinir. |
| /usr/lib/systemd/system/  | Dağıtım (package manager) tarafından sağlanan varsayılan birim dosyaları. En düşük öncelik seviyesindedir. |

12 farklı unit tipi mevcuttur:

| Unit Tipi   | Görevi |
|------------|--------|
| service    | Arka planda çalışan daemon veya program (sshd, httpd, ntp vb.) |
| target     | Birden fazla unit'i gruplayarak belirli bir sistem durumunu temsil eder (graphical.target, multi-user.target vb.) |
| timer      | Belirli aralıklarla veya zamanda servis çalıştırır (cron alternatifi) |
| mount      | Dosya sistemi mount noktalarını yönetir |
| socket     | Socket aktivasyonunu sağlar; istek geldiğinde servisi başlatır |
| device     | Donanım cihazlarını temsil eder |
| automount  | Dosya sistemine erişildiğinde otomatik mount yapar |
| swap       | Swap alanını yönetir |
| path       | Dosya sistemi değişikliklerini izler |
| scope / slice | Process gruplarını yönetir (cgroups) |

```code
systemctl list-unit
systemctl list-unit --type=service
systemctl list-unit-files

systemctl cat sshd-service
systemctl cat graphical.target

systemctl get-default
```

## systemctl - Servis Yönetimi

start vs enable = `enable --now`
`systemctl enable -now sshd`

`systemctl is-system-running` çıktısı ve anlamları:
| Durum        | Anlamı |
|-------------|--------|
| running     | Tüm unit'ler planlandığı şekilde çalışıyor |
| degraded    | En az bir unit başarısız; sistem çalışıyor ama eksik durumda |
| maintenance | Sistem bakım / rescue modunda |
| initializing | Boot süreci tamamlanmamış, sistem hazırlanıyor |
| starting    | Sistem başlatılıyor |
| stopping    | Sistem kapatılıyor |

## SysV Init — Geleneksel Init Sistemi

SysV'nin systemd'ye göre dezavantajları:
-	Servisleri **sıralı (sequential)** başlatır → yavaş boot
-	Servis bağımlılıklarını manuel script'lerle yönetmek gerekir
-	Paralel başlatma desteği yoktur
