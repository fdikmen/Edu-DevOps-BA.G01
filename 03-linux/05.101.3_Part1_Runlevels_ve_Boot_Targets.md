# 101.3 - Part 01 - Runlevels ve Boot Targets

Anahtar terimler: runlevel, boot target, systemctl isolate, telinit, init, /etc/inittab, /etc/rc[0-6].d, rescue, emergency, graphical.target, multi-user.target

##  Boot Target ve Runlevel Kavramı

Bir sistemin boot süreci boyunca geçtiği aşamalar şöyle düşünülebilir:
| No | Aşama | Açıklama |
|----|------|----------|
| 1 | Kernel yüklendi | Kernel belleğe alınır, temel donanım (CPU, RAM, disk) erişimi başlar |
| 2 | Root filesystem mount edildi | Ana dosya sistemi bağlanır ve sistemin kök dizini kullanılabilir hale gelir |
| 3 | Single-user / rescue modu | Sadece root erişimi vardır; bakım ve kurtarma işlemleri yapılır |
| 4 | Multi-user mode | Çok kullanıcılı çalışma ortamı; genellikle ağsız temel terminal erişimi |
| 5 | Network services | Ağ servisleri başlatılır, uzak bağlantılar (SSH vb.) mümkün olur |
| 6 | Graphical interface | Masaüstü ortamı (GUI) başlatılır |
| 7 | Ek servisler | Web server, DNS, database gibi kullanıcı servisleri aktif olur |
| 8 | Shutdown / reboot | Sistem kontrollü şekilde kapatılır veya yeniden başlatılır |

## systemd Boot Targets

En sık kullanılan target'lar ve karşılık geldikleri sistem durumları:
| Target            | Açıklama |
|------------------|----------|
| sysinit.target   | En temel sistem başlatma aşaması; çekirdek servislerin hazırlanması |
| basic.target     | Temel sistem bileşenleri aktif olur (cihazlar, soketler, zamanlayıcılar vb.) |
| multi-user.target | Çok kullanıcılı mod; ağ aktiftir, grafik arayüz yok (runlevel 3 eşdeğeri) |
| graphical.target | Çok kullanıcılı mod + grafik arayüz (runlevel 5 eşdeğeri) |
| rescue.target    | Tek kullanıcı modu; root erişimi, yerel diskler mount edilir, ağ yok |
| emergency.target | En minimal sistem; root filesystem read-only, neredeyse hiçbir servis yok |
| reboot.target    | Sistemi yeniden başlatır |
| halt.target      | Tüm süreçleri durdurur, CPU faaliyetlerini sonlandırır |
| poweroff.target  | Sistemi kapatır; ACPI üzerinden güç kesilir |

### Varsayılan Target'ı Görüntüleme ve Değiştirme
`systemctl get-default`
### Target Dosyasını İnceleme
`systemctl cat graphical.target`

### Target Bağımlılık Zinciri
graphical.target başlatma zinciri soldan sağa şu şekilde ilerler:
- sysinit.target → basic.target → multi-user.target → display-manager.service → graphical.target

#### Bağımlılık Tipleri
- **Requires (zorunlu bağımlılık)**  
  Bir unit olmadan hedef çalışmaz. Eksikse sistem hata verir veya hedef başlatılamaz.
- **Wants (isteğe bağlı bağımlılık)**  
  Unit yoksa bile sistem çalışmaya devam eder. Kritik değildir.

### systemctl isolate — Target'lar Arası Geçiş
*AllowIsolate=yes* olan target'lar arasında `systemctl isolate` komutu ile anlık geçiş yapılabilir. 

### Rescue ve emergency modları arasındaki farklar:

| Özellik | rescue.target | emergency.target |
|----------|--------------|------------------|
| Root FS durumu | Read-write mount edilmiş | Read-only mount edilmiş |
| Diğer diskler | Yerel diskler mount edilir | Hiçbir disk mount edilmez |
| Ağ bağlantısı | Yok | Yok |
| Kullanıcı erişimi | Yalnızca root | Yalnızca root |
| Ne zaman kullanılır? | Temel sistem kurtarma ve bakım | Kritik dosya sistemi veya boot hataları |

> Not: systemd, rescue, emergency, halt, poweroff ve reboot gibi kritik hedeflere geçiş sırasında sistemde oturum açmış tüm kullanıcılara otomatik olarak bir uyarı mesajı (wall mesajı) gönderir. 
Aynı uyarı mekanizması systemctl isolate komutu ile yapılan hedef değişikliklerinde de geçerlidir.

## SysV Runlevels

### Red Hat Tabanlı Runlevel'lar

| Runlevel | Açıklama |
|----------|----------|
| 0 | Sistem kapatma (Shutdown) — initdefault olarak ayarlanamaz |
| 1 (S/s) | Single-user mod: yalnızca root, kurtarma amaçlı, ağ yok |
| 2 | Çok kullanıcılı, ağ yok (NFS olmadan) |
| 3 | Çok kullanıcılı, tam ağ bağlantısı, grafik yok |
| 4 | Boş — sistem yöneticisi tarafından özelleştirilebilir |
| 5 | Çok kullanıcılı, tam ağ + grafik arayüz (X11) |
| 6 | Sistem yeniden başlatma (Reboot) — initdefault olarak ayarlanamaz |

### Debian / Ubuntu Tabanlı Runlevel'lar

| Runlevel | Açıklama |
|----------|----------|
| 0 | Sistem kapatma |
| 1 | Single-user mod |
| 2 | Çok kullanıcılı mod + grafik arayüz (Red Hat 5'e eşdeğer) |
| 3 | Sistem yöneticisi tarafından özelleştirilebilir |
| 4 | Sistem yöneticisi tarafından özelleştirilebilir |
| 5 | Sistem yöneticisi tarafından özelleştirilebilir |
| 6 | Sistem yeniden başlatma |

> Red Hat ve Debian sistemlerinde varsayılan çalışma seviyeleri farklıdır: Red Hat tabanlı sistemlerde grafik arayüz genellikle runlevel 5 üzerinden çalışırken, Debian tabanlı sistemlerde bu yapı runlevel 2 ile ilişkilendirilmiştir.

> SysV runlevel sistemi katı bir sıraya göre ilerler; sistem belirli bir seviyeye ulaşmak için önceki seviyelerden geçmek zorundadır.
systemd tarafında ise bu yapı değişmiştir.
Targets (hedefler) birbirine zincir şeklinde bağlı olmak zorunda değildir; her biri bağımsız olarak başlatılıp durdurulabilir ve ihtiyaçlara göre seçilebilir.


### SysV Runlevel Komutları (Eski sistemler / uyumluluk) : runlevel ve telinit

```
# SysV Runlevel Komutları

$ runlevel
N 5

$ sudo telinit 3   # terminal moda geçiş
$ runlevel
5 3

$ sudo telinit 5   # grafik moda dönüş

$ sudo init 0      # kapat
$ sudo init 6      # yeniden başlat

# Not
init ve telinit aynı işi yapar (systemd’de simüle edilir)
```


# SysV Runlevel & systemd (Özet)
- SysV sistemlerde sistemin çalışma modu **runlevel** ile belirlenir. Sistem açıldığında `init` süreci çalışır ve çalışma seviyesi `telinit` ile değiştirilir.

## Runlevel kontrolü

- $ runlevel
- Örnek: N 5
- - N → önceki runlevel yok (ilk boot)
- - 5 → mevcut durum (grafik mod)

----
## Runlevel değiştirme
  
- $ sudo telinit 3 
- Sistem terminal (multi-user) moda geçer.

----
## Değişiklik sonrası kontrol

- $ runlevel
- Örnek:5 3
- - 5 → önceki durum
- - 3 → yeni durum
----
## init komutu (alternatif)
- $ sudo init 3   # terminal mod
- $ sudo init 5   # grafik mod
- $ sudo init 0   # sistem kapatma
- $ sudo init 6   # reboot
----
## systemd karşılığı
- SysV runlevel yerine systemd “target” kullanır:
- telinit 3 → systemctl isolate multi-user.target
- telinit 5 → systemctl isolate graphical.target
----
- ## /etc/inittab (SysV yapı)
- /etc/inittab, eski Linux sistemlerinde (SysV init) sistemin nasıl açılacağını tanımlayan ana konfigürasyon dosyasıdır.
- Format:
- id:runlevels:action:process
- Anlamı:
    - id → satır adı (etiket gibi)
    - runlevels → hangi modlarda çalışacak (3,5 vs.)
    - action → nasıl çalışacak
    - process → çalıştırılacak komut
- Örnek:
- id:5:initdefault:
- Sistem açılışta grafik modda başlar.
----
- ## init.d servis yönetimi
- `init.d`, SysV init sisteminde servisleri başlatmak/durdurmak için kullanılan **script tabanlı yapı**dır.
- Her servis (ssh, apache, cron vb.) için `/etc/init.d/` içinde bir bash script bulunur.
- Bu scriptler “servis yöneticisi” gibi davranır.

- ### Temel kullanım

```bash
/etc/init.d/ssh start → servisi başlatır
/etc/init.d/ssh stop → servisi durdurur
/etc/init.d/ssh restart → kapatıp tekrar açar
/etc/init.d/ssh status → çalışıyor mu kontrol eder
```
- ### Modern karşılığı
```bash
systemd sistemlerinde bunun yerine:
systemctl start ssh
systemctl stop ssh
systemctl restart ssh
systemctl status ssh
```
----
- ## rc dizinleri nedir?
- `rc` dizinleri, SysV init sisteminde her runlevel için hangi servislerin çalıştırılacağını belirleyen klasörlerdir.
- `/etc/rcX.d/` (X = runlevel numarası) şeklinde bulunur.
- ### Örnek

```bash
$ ls /etc/rc5.d/
Örnek çıktı:
S10network
S15ssh
K01apache
```
```bash
Ne anlama gelir?
S → Start (servisi başlat) | K → Kill (servisi durdur)
Sayı → Çalışma sırası (priority)
Mantık => Küçük sayı → önce çalışır | Büyük sayı → sonra çalışır
Sistem açılırken veya kapanırken bu sıraya göre servisler yönetilir
YANI: rc dizinleri = runlevel’a göre servis başlatma/durdurma sırası
```
----
## SysV vs systemd farkı

| Özellik | SysV | systemd |
|----------|------|----------|
| Yönetim modeli | Runlevel | Target |
| Çalışma şekli | Sıralı (sequential) | Paralel (parallel) |
| Mimari | Katı ve eski yapı | Esnek ve modern yapı |
| Servis kontrolü | Script tabanlı (`/etc/init.d`) | `systemctl` tabanlı |
| Bağımlılık yönetimi | Zayıf / manuel | Güçlü dependency sistemi |
